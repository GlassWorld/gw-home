## 작업 범위

- 일일보고 목록에서 `작성` 진입 시 모달 대신 별도 페이지로 이동하도록 변경한다.
- 일일보고 작성 페이지를 `좌 3 : 우 7` 비율로 구성한다.
- 좌측에는 이전에 작성한 일일보고 목록을 배치한다.
- 우측에는 현재 작성 중인 일일보고 영역을 두고, 내부는 `작성 / 미리보기` 반반 구조를 유지한다.
- 개인 단위의 `Git 계정관리` 기능을 도입한다.
- Git 계정 하위에 프로젝트를 여러 개 등록 또는 관리할 수 있게 한다.
- 업무는 Git 계정정보를 직접 가지지 않고, 프로젝트 여러 개와 연결되도록 변경한다.
- `업무불러오기` 모달에서 선택한 업무 기준 프로젝트들의 커밋 목록을 Git provider API 로 조회해 선택 적용할 수 있게 한다.
- 프로젝트 연결이 없는 업무를 선택한 경우에는 커밋 조회 없이 업무명만 헤더에 세팅한다.

### 확정 정책

- 액세스 토큰은 **암호화하여 저장**한다.
- commit author 기준은 **이름(name)** 이다.
- 커밋 조회는 **Git provider API 기반** 으로 구현한다.
- provider 는 **선택형** 으로 설계한다.
- 사용자 하나에 Git 계정은 **복수 개** 를 허용한다.
- Git 계정 하나에 프로젝트는 **복수 개** 를 허용한다.
- 업무 하나에 프로젝트 연결은 **복수 개** 를 허용한다.

### 개념 모델

- 사용자 → Git 계정 여러 개
- Git 계정 → 프로젝트 여러 개
- 업무 → 프로젝트 여러 개

### 범위 제외

- 주간보고 화면 개편
- 관리자 Git 계정 관리 기능
- commit diff 상세 보기
- 브랜치 선택 기능
- 프로젝트 자동 동기화 고도화
- Git 설정 또는 프로젝트 연결이 없는 업무에 대한 강제 차단

## 완료 기준

- 개인 Git 계정관리에서 provider, author name, access token 을 여러 건 관리할 수 있다.
- Git 계정 하나에 프로젝트를 여러 건 연결할 수 있다.
- 업무등록에서 Git 토큰 직접 입력 없이 프로젝트 여러 건을 연결할 수 있다.
- access token 은 암호화 저장되고, 조회 응답에 원문이 노출되지 않는다.
- 프로젝트 연결 기반 Git commit 조회 API 가 작성일 + author name 기준으로 동작한다.
- 일일보고 목록에서 작성 진입 시 모달이 아니라 별도 페이지로 이동한다.
- 작성 페이지는 좌측 이전 일일보고 목록, 우측 작성/미리보기 구조로 동작한다.
- 업무불러오기 모달에서 선택 업무 기준 프로젝트 커밋을 불러와 일일보고에 반영할 수 있다.
- 프로젝트 연결이 없는 업무는 업무명만 헤더로 세팅되고 수동 작성이 가능하다.
