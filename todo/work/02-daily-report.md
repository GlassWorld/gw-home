## 작업 순서

1. 상태 모델을 `activeTask`와 `selectedTaskUuids` 중심으로 재정의한다.
2. 작성 모달을 3분할 레이아웃으로 바꾸고 상위 페이지가 상태를 소유하게 정리한다.
3. 좌측 업무 목록 패널을 먼저 분리하고 검색/선택 흐름을 완성한다.
4. 중앙 과거 보고 패널을 기존 데이터 재가공 또는 더미 데이터 기반으로 연결한다.
5. 우측 작성 패널을 구조화 UI로 바꾸고 저장 시 현재 API 스펙에 맞게 조합한다.
6. skeleton, empty state, 선택 강조 등 UX 마감 작업을 한다.
7. 추가 API가 필요한 부분은 2차 작업 메모로 분리한다.

## 완료 기준

- 작성 모달이 좌/중/우 3분할 구조로 동작한다.
- 좌측에서 업무를 선택하면 중앙 히스토리와 우측 작성 패널이 함께 반응한다.
- 패널이 컴포넌트 단위로 분리되어 있다.
- 데스크탑 기준에서 각 패널이 독립 스크롤 가능하다.
- 로딩, 빈 상태, 선택 상태가 명확하게 표현된다.
- 현재 일일보고 저장 구조와 호환되게 저장/수정이 가능하다.
- 추가 API가 필요한 항목은 문서에 후속 범위로 분리되어 있다.
