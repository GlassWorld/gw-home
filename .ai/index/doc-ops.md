# DOC OPS INDEX

This document guides review and maintenance work for the AI document system itself.

## When To Use

- Reviewing `.ai` document quality
- Restructuring entry, index, or work document flows
- Auditing token efficiency, duplication, or reference accuracy
- Planning skillization for repeated document workflows

## Minimum Reading Set

- `.ai/AGENTS.md`
- `.ai/CORE_RULES.md`
- `.ai/index/routing.md`
- `.ai/index/work-docs.md`
- `.ai/index/skills.md`

Read `.ai/project/summary.md` only if the change depends on project stack or module structure.

## Review Focus

- Entry cost: how many files must be read before useful work starts
- Reference accuracy: whether documented paths exist
- Responsibility split: whether one rule lives in one place
- Token efficiency: whether the same guidance is repeated across documents
- Execution support: whether triggers and outputs lead to immediate action

## Skillization Heuristics

- Convert repeated output-writing workflows into skills
- Keep absolute rules as documents
- Keep project facts and module summaries as documents
- Use skills for classification or formatting steps that repeat often
