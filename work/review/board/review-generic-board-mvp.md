# 요청 작업 검토 문서

## Scope

- 대상:
  - 범용 게시판 기능 요구사항을 현재 `board/comment/file` 구현 위에 확장하는 작업
- 관점:
  - 기존 구조와 충돌 최소화
  - 이번 턴에 끝낼 수 있는 MVP 범위 우선
  - 후속 구조 개편이 필요한 항목 분리

## Summary

- 한줄 요약:
  - 현재 저장소는 게시글/댓글/파일의 기본 축은 있으나, 운영형 게시판으로 보기에는 작성 화면, 마크다운 경험, 첨부 관리, 검색, 권한 표현이 부족하다.
- 전체 판단:
  - 완전한 범용 게시판 리모델링은 HEAVY 범위다.
  - 다만 기존 구조를 유지하면서도 체감 기능을 크게 높이는 1차 MVP 구현은 이번 작업으로 진행 가능하다.

## Findings or Scores

- 항목:
  - 현재 게시글 구조
- 점수:
  - 3/5
- 근거:
  - 게시글 CRUD, 목록, 상세 API는 있으나 게시판 단위 설정과 작성/수정 화면이 부족하다.
- 영향:
  - 최소 게시판 기능은 가능하지만 운영형 확장성은 낮다.

- 항목:
  - 댓글/답글 구조
- 점수:
  - 3/5
- 근거:
  - 부모 댓글 연결은 이미 있으나 depth 제한과 프론트 사용성이 없다.
- 영향:
  - 답글은 데이터상 가능하지만 정책 통제가 약하다.

- 항목:
  - 파일/첨부 구조
- 점수:
  - 2/5
- 근거:
  - 독립 파일 업로드는 있으나 게시글과의 연결, 첨부 목록, 다운로드 흐름이 없다.
- 영향:
  - 본문 이미지와 일반 첨부를 운영 관점에서 구분 관리하기 어렵다.

- 항목:
  - 마크다운 및 보안
- 점수:
  - 3/5
- 근거:

- 영향:
  - 본문 작성/렌더링 경험이 현재는 plain text 수준에 머물러 있다.

## Recommended Next Actions

1. 게시글 작성/수정 화면, 마크다운 미리보기, 댓글/답글 UI를 1차 MVP로 구현한다.
2. 게시글 검색 조건 확장, 카테고리 조회, 첨부 연결 구조를 백엔드에 추가한다.
3. 게시판 마스터 분리와 세부 권한 정책은 후속 단계로 남기고, 이번 턴에는 기존 카테고리 구조를 재사용한다.
