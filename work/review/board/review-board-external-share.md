# 게시판 외부 공유 기능 검토

## Scope

- 대상: 게시글 외부 공유 링크 생성, 만료 검증, 읽기 전용 공개 페이지
- 관점: 보안성, 기존 게시글 상세와의 분리, 풀스택 구현 영향

## Summary

- 한줄 요약: 게시글 외부 공유는 별도 토큰 저장 구조와 공개 전용 조회 흐름을 추가하는 풀스택 기능이다.
- 전체 판단: 기존 게시글 상세 화면을 재사용하지 않고 공유 전용 API와 전용 페이지를 분리하는 방향이 요구사항에 가장 부합한다.

## Findings or Scores

- 항목: 현재 게시글 상세 API는 인증 여부와 무관하게 내부 상세 데이터를 반환한다.
- 점수: 중간 위험
- 근거: 기존 `/api/v1/boards/{boardPostUuid}` 조회는 공유 상태나 외부 전용 제한 없이 일반 상세 응답을 제공한다.
- 영향: 외부 공유는 기존 상세 API 재사용이 아니라 별도 공개 조회 API가 필요하다.

- 항목: 현재 보안 설정에는 공유 토큰 기반 공개 접근 경로가 없다.
- 점수: 중간 위험
- 근거: `SecurityConfig` 는 기존 게시글 상세와 댓글 공개 조회만 허용하며, 공유 검증 흐름은 없다.
- 영향: 공유 토큰 조회, 비밀번호 확인, 상태 코드 구분을 위한 공개 엔드포인트와 허용 규칙이 필요하다.

- 항목: 현재 게시판 프론트는 상세 화면 안에 수정, 삭제, 댓글, 좋아요 상호작용이 강하게 결합되어 있다.
- 점수: 높은 영향
- 근거: 게시글 상세 페이지가 댓글 작성/수정/삭제, 첨부 다운로드, 좋아요까지 한 화면에서 제공한다.
- 영향: 외부 공유는 별도 페이지와 별도 타입/컴포저블로 분리해야 읽기 전용 정책을 안정적으로 지킬 수 있다.

## Recommended Next Actions

1. 공유 토큰 저장 테이블과 만료/비밀번호 정책을 정의한다.
2. 내부 관리 API와 외부 공개 조회 API를 분리해 구현한다.
3. 게시글 상세의 공유 설정 모달과 외부 전용 페이지를 각각 구현하고 읽기 전용 UX를 고정한다.
