# 관리자 OTP 선택 기능 계획

## Goal

- 목적:
  - 관리자가 계정별 OTP 요구 여부를 선택할 수 있게 하고, 해당 선택이 로그인 정책에 반영되도록 수정한다.

## Scope

- 포함:
  - 계정 OTP 정책 필드 추가와 계정 모델 반영
  - 관리자 계정 상세/수정 API 확장
  - 관리자 계정 관리 모달에서 OTP 정책 선택 UI 추가
  - 로그인 시 OTP 정책 분기 수정
  - 관련 단위 테스트와 프런트 타입 보강
- 제외:
  - 기존 계정 데이터 일괄 마이그레이션 스크립트 실행
  - OTP QR 등록 UX 전면 개편
  - 일반 사용자 설정 화면 문구 재디자인

## Current State

- 현재 상태:
  - `otp_enabled` 하나로 OTP 등록 상태와 로그인 강제 상태를 함께 표현한다.
  - 관리자는 OTP 실패 초기화와 OTP 재등록 초기화만 수행할 수 있다.
  - 로그인은 OTP가 비활성화된 계정에 대해 `OTP_SETUP_REQUIRED`를 반환한다.
- 제약:
  - DB 필드 추가가 필요할 가능성이 높다.
  - 로그인 정책 변경은 전 계정 로그인에 영향을 주므로 테스트가 필수다.

## Action Items

1. 계정 정책 필드와 Mapper/VO/응답 DTO 구조를 설계하고 반영한다.
2. 관리자 계정 API와 관리자 화면에서 OTP 정책 조회/수정 기능을 추가한다.
3. 로그인, OTP 상태 조회, 관련 단위 테스트를 정책 분기에 맞게 갱신한다.

## Done Criteria

- 관리자가 계정별 OTP 요구 여부를 변경할 수 있다.
- OTP 요구가 꺼진 계정은 OTP 등록 없이 일반 로그인에 성공한다.
- OTP 요구가 켜진 계정은 미등록 시 등록 요구, 등록 완료 시 OTP 인증 흐름으로 진입한다.
- 관리자 계정 관리와 로그인 관련 테스트가 통과한다.
