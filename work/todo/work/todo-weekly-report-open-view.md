# 주간보고 공개 조회 기능 계획

## Goal

- 목적: 주간보고 메뉴를 `내 주간보고`, `인별 주간보고 조회`로 분리하고, 공개된 주간보고를 사용자별로 조회할 수 있게 만든다.

## Scope

- 포함:
  - 주간보고 메뉴/화면 분리
  - 공개된 주간보고 목록 조회 API 추가
  - 사용자 기준 필터 또는 사용자 선택 UI 추가
  - 공개 주간보고 상세 조회 흐름 추가
- 제외:
  - 주간보고 DDL 변경
  - 권한 체계 전면 개편
  - 비공개 보고 공유 기능

## Current State

- 현재 상태:
  - `/work/weekly-reports` 화면은 내 주간보고 작성/수정/목록만 제공한다.
  - `openYn`과 `publishedAt`은 저장되지만 공개 보고를 타인이 조회하는 API와 UI는 없다.
- 제약:
  - 백엔드 응답과 요청은 `snake_case` 계약을 따라야 한다.
  - 기존 내 주간보고 작성/수정 흐름은 유지되어야 한다.

## Action Items

1. 공개 주간보고 목록/상세 조회 백엔드 API와 Mapper 쿼리를 추가한다.
2. 프론트 주간보고 화면을 `내 주간보고`와 `인별 주간보고 조회` 섹션 또는 탭 구조로 분리한다.
3. 사용자 선택과 공개 보고 상세 확인 UI를 연결하고 기존 작성/수정 흐름 회귀를 점검한다.

## Done Criteria

- 내 주간보고와 인별 주간보고 조회가 화면에서 분리되어 보인다.
- 공개된 주간보고만 다른 사용자 기준으로 조회된다.
- 기존 주간보고 생성, 수정, 공개 여부 저장이 그대로 동작한다.
