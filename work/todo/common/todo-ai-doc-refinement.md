# .ai 문서 세팅 후속 정리 계획

## 목적

- `.ai` 문서 세팅의 바이브코딩 활용도를 더 높인다.
- 토큰 재소모를 줄이고 문서 간 source of truth를 더 분명하게 만든다.
- `##검토`, `##계획`, `##작업`, `##커밋` 흐름을 실사용 기준으로 더 안정화한다.

## 범위

- 포함:
  - `.ai/AGENTS.md`
  - `.ai/TASK_ROUTER.md`
  - `.ai/SKILL_INDEX.md`
  - `work/review/`
  - `work/todo/`
- 제외:
  - 애플리케이션 코드 변경
  - 도메인별 개발 문서 본문 개편

## 현재 상태

- 코어 문서 중복은 1차 정리 완료
- `summary`는 요약 역할 위주로 축소 완료
- skill 해석 규칙은 `SKILL_INDEX` 중심으로 정리 완료
- 아직 `##계획`과 `##검토` 산출물 형식은 표준 템플릿이 약하다

## 할 일

1. `##검토` 문서 기본 템플릿 정의
2. `##계획` 문서 기본 템플릿 정의
3. `AGENTS`의 트리거 설명을 더 짧고 일관된 형식으로 정리
4. `TASK_ROUTER`에 work 문서 작성 최소 항목을 명시
5. review / todo 문서 네이밍 규칙 예시를 보강

## 산출물

- `##검토` 템플릿 초안
- `##계획` 템플릿 초안
- 문서 트리거 설명 정리본
- work 문서 최소 작성 규칙

## 완료 기준

- 같은 규칙이 2개 이상 문서에 중복 설명되지 않는다
- 새 작업 요청이 들어왔을 때 `AGENTS -> TASK_ROUTER -> work 문서` 흐름이 자연스럽다
- `##검토`와 `##계획` 결과물이 문서마다 크게 흔들리지 않는다

## 메모

- 다음 `##작업` 요청이 오면 이 문서를 기준으로 실제 템플릿 정리 작업 진행
